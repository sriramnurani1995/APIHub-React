API Endpoints

Our API offers the following endpoints:

Endpoint Description
GET /api/weather/date/:date? Retrieve weather data for a specific date. Defaults to the current date if not specified.
GET /api/weather/month/:month? Retrieve weather data for a specific month. Defaults to the current month if not specified.

Data Fields

The API returns the following fields in its responses:

Field Description Example
date The date for which the weather data is generated. 2024-12-06
temperature The main temperature for the day in Celsius. 12
minTemperature The minimum temperature for the day. 8
maxTemperature The maximum temperature for the day. 16
precipitation The amount of precipitation in millimeters. 5
wind The wind speed in km/h. 15
gusts The wind gust speed in km/h. 25
condition The overall weather condition. Cloudy
description A brief description of the weather condition. Overcast skies with mild temperatures.

Realistic Randomization

Our API uses advanced algorithms to ensure realistic randomization. Here’s how we make the data believable:

  • Temperature: Adjusted for seasons, with appropriate min and max ranges for each day.
  • Conditions: Weather conditions like "Snowy" are tied to temperatures at or below 0°C.
  • Precipitation: Seasonal likelihoods for rain or snow ensure accuracy.
  • Wind and Gusts: Dynamically generated, with stronger gusts in winter months.
